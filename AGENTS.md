# WoW 插件工作空间协作说明

本工作空间用于编写多个不同的魔兽世界插件，不同会话可能分别处理不同插件。

## 仓库信息

- GitHub 仓库：`https://github.com/xingxp/wow-plugins`
- 默认分支：`main`
- 新机器继续工作时，先 clone 这个仓库，再根据本机 WoW 安装路径同步对应插件目录。

```bash
git clone https://github.com/xingxp/wow-plugins.git
cd wow-plugins
```

当前仓库内已有插件：

- `BigMeteor`：好大的流星，毁灭术火/暗印记与大流星提示。
- `DKGunShuBiao`：DK滚鼠标，显示鼠标滚轮触发的动作按钮。

## 会话接管规则

- 进入工作空间后，先确认“当前会话正在处理哪个插件”。
- 不要默认 `dkm-plugin-handoff.md` 适用于所有会话；它只代表 `DKGunShuBiao` 这个独立插件的交接记录。
- 如果用户没有明确当前插件名，先从当前目录结构、目标文件夹名、`.toc` 文件、用户描述中确认插件范围。
- 优先把“跨插件通用经验”写入本工作空间文档，把“单插件特有背景”写入对应插件自己的交接文件。

## 目录约定

- 仓库根目录可以存放工作空间级文档、脚本、经验总结。
- 每个插件最好有自己独立的目录，目录名尽量使用 ASCII。
- 每个插件目录至少应包含：
  - `*.toc`
  - 一个或多个 `*.lua`
  - 可选的 `media/`
  - 可选的该插件专属交接文档

## 通用开发原则

- 默认写独立插件，不优先依赖 WeakAuras 作为最终交付形态。
- 不轻易接管用户输入、覆盖原始绑定或修改用户宏，除非用户明确要求。
- 先兼容现有 UI 环境，再考虑理想化实现；常见环境包括暴雪动作条、NDui、LibActionButton。
- 新建或更新插件 `.toc` 时，作者默认填写为 `王老狮@HC`，除非用户明确指定别的作者。
- SavedVariables 和插件目录命名尽量稳定、可迁移、可回滚。
- 新增调试输出时，优先做成可开关，而不是常驻刷屏。

## 修改前检查

- 确认目标客户端目录和插件安装目录。
- 确认当前接口版本，避免沿用过时 `Interface` 值。
- 确认本次修改针对的是哪个插件，而不是误改到另一个插件。
- 如果某份文档明显只属于单插件，不要把它当成全局真相。

## 修改后检查

- 至少做一次 Lua 语法加载校验。
- 如果本次修改的是插件源码，记得同步更新该插件的版本号，例如更新 `.toc` 里的 `## Version`。
- 如果本次修改的是插件源码，默认还要把最新文件同步到对应游戏 `Interface/AddOns/<PluginDir>` 目录，不把“手动拷贝”留给用户。
- 如果工作空间里已有插件源码目录，默认以工作空间目录为主进行修改，不直接把游戏目录当成唯一源码。
- 如果有插件专属验证步骤，把验证步骤记录到该插件自己的文档里。
- 如果这次产出了可复用经验，把经验补充到 `docs/wow-addon-playbook.md`，而不是只留在聊天记录里。

## 文档分层

- `AGENTS.md`：工作空间级协作规则，给未来所有会话看。
- `docs/wow-addon-playbook.md`：跨插件通用经验、调试方式、兼容性结论。
- `*-handoff.md`：某个具体插件的背景、约束、现状和待办。
