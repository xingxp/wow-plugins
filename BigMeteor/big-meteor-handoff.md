## 当前行为

- 左侧红色竖条表示目标 `火` 印记层数
- 右侧紫色竖条表示目标 `暗` 印记层数
- 两个计量条靠近显示，作为核心状态区
- 两侧计量条按层数从底部向上点亮
- 火/暗印记的倒计时数字直接压在各自计量条中间显示
- 右侧显示两行推荐轨道，格式为图标 + 技能名称
- 上方轨道在玩家读条/引导时变成当前施法进度条，类似卡拉 OK 时间线
- 如果当前施法匹配插件开始施法时的主推荐，进度条显示绿色；如果不匹配，显示红色，方便判断推荐逻辑和实战手感是否错位
- 下方轨道显示下一步预告；读条时会跳过当前正在施放的技能，尽量显示真正下一步
- 0.5.25 起当前施法期间会锁定下一手候选，目标是让第二行成为稳定队列：玩家按第一行施法时可以提前看第二行准备，当前技能结束后第二行应自然上移成为第一推荐
- 锁定候选只允许被更高优先级的暗灼/流星/燃烧/献祭抢占，避免普通填充因 CD/层数 0.1 秒重算而频繁跳变
- 0.5.26 起第二行不再只是“当前状态第二名”，而是当前技能成功落地后的预演下一手：暗灼预演为燃烧/流星，流星预演为混乱箭，献祭预演为燃烧，燃烧预演为暗灼/流星或混乱箭
- 预演队列的目标是让玩家按第一行施法时，第二行已经是技能完成后的正确第一推荐，而不是读条过程中才重算跳变
- 0.5.21 修复当前施法轨道与下一推荐轨道可能显示同一个技能的问题：跳过下一推荐时同时按 spellID 和技能名匹配，兼容客户端读条 API 不返回稳定 spellID 的情况
- 当前推荐逻辑已经从“即时优先级”改为“暗灼 + 流星窗口调度器”
- 调度器根据高手日志抽象出约 16 秒主循环：`暗影灼烧 -> 混沌流星 -> 混乱之箭`
- 调度器先预测下一次暗影灼烧 + 混沌流星窗口，再决定当前技能是否能插入，避免读条拖慢窗口
- 0.5.20 起加入燃烧 6 秒流星质量窗口：流星不只追求双 6，还尽量落在玩家刚释放燃烧后的 6 秒内
- 如果暗灼 + 流星窗口已经成立，且燃烧可用、有玩家自己的献祭、插入燃烧后火/暗层仍能撑到流星落点，会先推荐燃烧，再推荐暗影灼烧/混沌流星
- 如果暗灼后已经进入流星窗口，燃烧可用且插入后流星仍能吃到双层，会先推荐燃烧，再推荐混沌流星
- 0.5.22 起暗影灼烧成功后进入强链式承诺：短窗口内推荐只围绕 `燃烧 -> 混沌流星` 或 `混沌流星`，不会因为目标光环刷新延迟、层数预测抖动或流星短暂 GCD/CD 抖动而把混沌流星从列表移除
- 0.5.24 加强暗影灼烧识别：除了 `UNIT_SPELLCAST_SUCCEEDED` 的 spellID，也会用施法开始记录和施法停止兜底来进入流星链，避免客户端事件参数不稳定时暗灼后掉回暗影箭
- 如果燃烧马上转好，且等待不会导致流星窗口失败，会推荐生命分流等待燃烧窗口
- 进入窗口条件：火层能撑到流星落点，暗影灼烧提供 `+2 暗层` 后，暗层当前或读条期间能到 6，且混沌流星可用
- 玩家刚释放暗影灼烧后进入短暂流星窗口；此时混沌流星默认最高，但如果燃烧已经可用且插入后流星落点仍能吃到双层，允许先插燃烧
- 如果暗影灼烧即将可用，且混乱箭/烧尽/暗影箭等读条会跨过暗灼 CD，推荐生命分流等待窗口
- 献祭按 `1.5s / (1 + 当前急速)` 动态计算预读时间，并加 `0.6s` 安全提前量；进入预读窗口后会优先推荐献祭，目标是读条过程中旧献祭刚好消失，新献祭落地无缝接上
- 献祭预读不会打断“暗影灼烧成功后的强制流星链”，但在非强链状态下优先级高于燃烧/混乱箭/普通填充，避免等到献祭消失才提示
- 燃烧只有目标身上存在玩家自己的献祭时才会推荐
- 混乱之箭优先作为流星后的高价值读条或安全插入技能，不再无条件压过窗口准备
- 暗影烈焰按立即 `+2 火层 +2 暗层` 计算；如果一发暗影烈焰能把下一次暗灼窗口准备好，会推荐暗影烈焰
- 暗影烈焰需要 11 码距离；如果目标不在暗影烈焰范围内且需要补暗层，推荐暗影箭
- 暗不足时提高暗影箭/暗影烈焰权重，烧尽主要用于火层不足或普通填充
- 没有目标时保持空条，不额外提示
- 读取层数时同时兼容 `C_UnitAuras` 和 `UnitDebuff` 两套路径，优先为木桩目标保证能读到 debuff 层数
- 只统计玩家自己施加的火/暗印记、献祭、暗影烈焰；团队里其他术士叠的同名 debuff 不参与推荐计算
- 对于“debuff 已存在但层数字段返回 0/nil”的情况，按至少 1 层处理，并在整列 debuff 中取最大层数
- 临时在插件下方打印目标前 12 个 debuff 的 `spellID:名字 x层数`，用于锁定暗印记真实数据
- 插件启动时强制使用代码内置的火/暗印记 ID，避免旧 SavedVariables 污染当前读取
- 支持拖动位置
- 支持按角色保存配置

## 施法轨道设想

右侧推荐区的长期方向不是“静态告诉玩家下一个技能”，而是做成可追踪执行的施法轨道：

- 上方轨道表示当前正在执行的技能；玩家开始读条/引导后，推荐卡片变成横向时间线，像卡拉 OK 字幕一样跟随施法进度推进
- 下方轨道表示下一步预告；当前技能正在释放时，插件仍然继续计算下一手，减少读条结束后才反应的滞后感
- 施法开始瞬间记录“当时插件推荐的主技能”，用于判断玩家是否按推荐执行，而不是事后用新的推荐结果倒推
- 如果当前施法匹配推荐，进度条用绿色；如果不匹配，用红色，方便回看是玩家临场调整，还是推荐逻辑确实不合实战
- 后续可以基于这套轨道继续做“预演队列”：例如献祭读条期间预测献祭落地、燃烧变为可用、暗影烈焰 DOT 跳暗层，从而更早给出下一手
- 这套机制的核心价值是帮助调优推荐算法：当红色经常出现，说明插件推荐和玩家判断有分歧，应优先分析当时的火/暗层数、CD、射程、献祭剩余和团队内其他术士 debuff 干扰
