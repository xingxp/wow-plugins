# WoW 插件开发经验手册

更新时间：2026-05-02

这份手册记录的是跨插件可复用的经验，不绑定某一个具体插件。

## 1. 先分清“工作空间经验”和“单插件背景”

- 工作空间里可能同时维护多个插件。
- 某个 `*-handoff.md` 往往只对应一个插件，不应自动推广到其它插件。
- 如果结论只在某个插件、某个职业、某个 UI 环境成立，应写在插件专属文档中。
- 如果结论对多个插件都有价值，再沉淀到这份手册。

## 2. 插件目录与命名

- 插件目录名优先使用 ASCII，避免路径和资源加载兼容性问题。
- 插件列表显示名可以使用中文。
- 资源路径应使用标准 WoW 路径格式，例如：

```lua
"Interface\\AddOns\\MyAddon\\media\\icon.tga"
```

## 2.1 换机器继续工作

- 仓库地址：`https://github.com/xingxp/wow-plugins`
- 新机器先 clone 仓库，再把目标插件目录同步到本机 WoW AddOns 目录。
- 不要把游戏目录当作唯一源码；仓库里的插件目录是主源码。
- 如果新机器 WoW 路径不同，只需要调整同步目标路径，不需要改插件内资源路径。
- 提交前确认没有把 `tmp/`、`.DS_Store`、解码临时文件放进 git。

## 3. 先做独立插件，再考虑 WeakAuras

- 当需求需要稳定监听按钮点击、绑定来源、动作槽、事件流时，独立插件通常比 WA 更可靠。
- WA 适合快速验证 UI 表现、固定槽位展示、轻量提示。
- 如果需求涉及“不能改宏、不能接管输入、还要精确知道触发来源”，优先评估独立插件。

## 4. 输入和绑定相关的经验

- 不要轻易使用覆盖绑定去接管用户输入，尤其是滚轮。
- 覆盖绑定可能会抢到输入，但不一定能安全、完整地转发回原动作。
- 在部分客户端环境里，滚轮一旦被接管，容易出现“输入被截获但动作没触发”的问题。
- 如果只是识别“哪个鼠标绑定触发了动作”，优先读取现有绑定和按钮信息，而不是改写绑定。

## 5. 动作条兼容思路

- 不要假设所有动作按钮都来自暴雪默认动作条。
- 常见兼容对象包括：
  - 暴雪动作条按钮
  - NDui 动作条
  - `LibActionButton-*`
- 读取按钮槽位时，不要只读一个字段；应准备多种候选字段。
- 读取绑定目标时，也不要只信任一个字段或一个点击目标名。

常见可尝试的数据来源：

```lua
button.action
button._state_action
button:GetAttribute("action")
button:GetAttribute("action1")
button:GetAttribute("labaction-0")
button:GetAttribute("labaction-1")
```

常见绑定目标候选：

```lua
button.keyBoundTarget
button.bindName
button.config and button.config.keyBoundTarget
button:GetName()
"CLICK " .. name .. ":Keybind"
"CLICK " .. name .. ":LeftButton"
```

## 6. Hook 策略

- 优先 hook 真实按钮，而不是只依赖全局兜底函数。
- 对动态创建的按钮，要考虑注册库回调或周期性补扫。
- 兜底 hook 可以保留，但应建立在“目标槽位已被识别”的前提下，避免误报。
- 如果某个实现依赖外部 UI 插件创建按钮，通常需要同时支持：
  - 初始扫描
  - 延迟扫描
  - 动态创建回调

## 7. 调试方式

- 优先做一个可开关的调试模式，例如斜杠命令开关。
- 调试输出应包含：
  - 当前识别到的槽位
  - 来源路径
  - 绑定键或方向
- 避免默认持续刷屏。

## 8. 验证方式

- 每次改 Lua 后，至少做一次 `loadfile` 级别的语法校验。
- 每次改插件源码后，顺手更新该插件 `.toc` 里的版本号，避免游戏里看不出当前构建。
- 每次改插件源码后，默认把最新文件同步到游戏 `Interface/AddOns/<插件目录>`，让用户只需要 `/reload`。
- 如果工作空间内已经有该插件的源码副本，应优先修改工作空间副本，再同步到游戏目录。
- 语法校验只能证明“能加载”，不能证明“游戏行为正确”。
- 真正涉及动作条、绑定、职业资源条、战斗状态时，仍需要进游戏 `/reload` 验证。

推荐最小校验套路：

```bash
lua -e 'assert(loadfile("/绝对路径/YourAddon.lua"))'
```

## 9. 文档怎么写

- 插件专属背景写进对应插件的交接文件。
- 跨插件结论写进本手册。
- 让后续会话一眼看懂比写得“完整但混在一起”更重要。

推荐结构：

1. 这个插件要解决什么问题
2. 当前部署位置
3. 当前文件
4. 运行环境
5. 已知约束
6. 验证命令
7. 游戏内验证步骤
8. 后续待办
